Documentação da API Escola

API RESTful para cadastro de alunos, disciplinas e matrículas.

Rotas de Alunos

Método Rota Propósito Parâmetros obrigatórios Exemplo de saída
GET /alunos Lista todos os alunos - [ { "id": 1, "nome": "João" }, ... ]
POST /alunos Cria um novo aluno nome (string) { "id": 4, "nome": "Carlos" }
PUT /alunos/:id Atualiza o nome do aluno nome (string) { "id": 2, "nome": "Maria Silva" }
DELETE /alunos/:id Remove o aluno e suas matrículas - 204 No Content

Rotas de Disciplinas

Método Rota Propósito Parâmetros obrigatórios Exemplo de saída
GET /disciplinas Lista todas as disciplinas - [ { "id": 1, "nome": "Matemática" }, ... ]
POST /disciplinas Cria uma nova disciplina nome (string) { "id": 4, "nome": "Geografia" }
PUT /disciplinas/:id Atualiza o nome da disciplina nome (string) { "id": 2, "nome": "Português Avançado" }
DELETE /disciplinas/:id Remove a disciplina e suas matrículas - 204 No Content

Rotas de Matrículas

Método Rota Propósito Parâmetros obrigatórios Exemplo de saída
GET /matriculas Lista todas as matrículas - [ { "id": 1, "alunoId": 1, "disciplinaId": 1, "nota": 8.5 }, ... ]
POST /matriculas Cria uma matrícula alunoId (int), disciplinaId (int), nota (float, 0-10) { "id": 4, "alunoId": 2, "disciplinaId": 3, "nota": 7.0 }
PATCH /matriculas/:id Atualiza a nota da matrícula nota (float, 0-10) { "id": 2, "nota": 9.5 }
DELETE /matriculas/:id Remove a matrícula - 204 No Content

Observações

  • Todos os endpoints retornam JSON.
  • Parâmetros obrigatórios devem ser enviados no corpo da requisição (POST, PUT, PATCH).
  • IDs são inteiros autoincrementados.
  • Ao remover aluno ou disciplina, todas as matrículas relacionadas também são removidas.
  • Notas devem ser números entre 0 e 10.
